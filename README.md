# 🎬 سیستم رزرو بلیط سینما – Cinema Booking System

یک **پروژه سطح متوسط** برای رزرو آنلاین بلیط سینما ساخته شده با **JavaScript خام (Vanilla JS)** و **HTML/CSS**. این پروژه نمایشگر **مهارت‌های فرانت‌اند** است.

## 🎯 توضیح پروژه

این پروژه یک **رابط کاربری کامل و تعاملی (Interactive UI)** برای سیستم رزرو بلیط سینما است که تنها با **Vanilla JavaScript، HTML5 و CSS3** پیاده‌سازی شده است. بدون استفاده از فریم‌ورک‌های پیچیده، برای نمایش توانایی در ساخت یک اپلیکیشن واقعی با جاوااسکریپت خام.

## 🛠️ تکنولوژی‌ها

### فرانت‌اند (تنها قسمت پروژه)
- **HTML5** – ساختار صفحات و المان‌های سمانتیک
- **CSS3** – استایل‌دهی مدرن با:
  - Grid و Flexbox برای طراحی ریسپانسیو
  - CSS Variables برای رنگ‌ها و متغیرهای استایل
  - Animations و Transitions برای تجربه کاربری نرم
  - Media Queries برای سازگاری با دستگاه‌های مختلف

- **Vanilla JavaScript (ES6+)** – منطق کامل بدون فریم‌ورک:
  - DOM Manipulation برای تعاملات پویا
  - Event Handling برای رسیدگی به رویدادها
  - Local Storage برای ذخیره‌سازی اطلاعات کاربر
  - Async/await و Fetch API برای شبیه‌سازی درخواست‌های API
  - Modules (ES6) برای تنظیم کد

### Dark Mode & RTL
- ✅ طراحی **راست‌به‌چپ (RTL)** کامل برای فارسی
- ✅ **تم تیره** مدرن برای تجربه سینمایی

## 🚀 نحوه راه‌اندازی

### پیش‌نیازها
- درایور مرورگر (Chrome, Firefox, Safari, Edge)
- متن‌ویرایشگر یا IDE (VS Code)
- Live Server یا هر سرور فایل استاتیک

### مراحل نصب و اجرا

**گزینه 1: استفاده از Live Server (VS Code)**
1. پروژه را کلون کنید:
```bash
git clone https://github.com/your-username/cinema-tiket.git
```
2. پوشه پروژه را در VS Code باز کنید
3. روی `index.html` کلیک راست کنید و **"Open with Live Server"** را انتخاب کنید

**گزینه 2: استفاده از یک سرور ساده**
```bash
cd cinema-tiket

# اگر Python 3 دارید:
python -m http.server 8000

# یا اگر Node.js دارید:
npx http-server
```
سپس در مرورگر به `http://localhost:8000` بروید.

**گزینه 3: باز کردن مستقیم در مرورگر**
- فایل `index.html` را در مرورگر باز کنید (این روش محدودیت‌های Cross-Origin دارد)

## 📊 شبیه‌سازی API و Local Storage

این پروژه تماماً **فرانت‌اند است** و با استفاده از:
- **Local Storage** برای ذخیره‌سازی داده‌های کاربر (ورود، علاقه‌مندی‌ها، کارت‌های صندلی)
- **Mock Data** (داده‌های ساختگی) برای نمایش فیلم‌ها، سینماها و سانس‌ها
- **داده‌های استاتیک** برای شبیه‌سازی رفتار یک بک‌اند واقعی

> **توجه:** برای اتصال به یک بک‌اند واقعی (API server)، می‌توان کد JavaScript را با Fetch API و endpoints واقعی به‌روزرسانی کرد.
